El estudiante debe aprender a instalar un nuevo sistema operativo de red sobre otro ya existente, salvaguardando la integridad de la información y configurando actualizaciones automáticas de software y seguridad.



---

## Plan de Clases

### Clase 1: Preparación para la Instalación de un Nuevo Sistema Operativo (6 horas)

**Objetivo**:  
Preparar el entorno para la instalación de un nuevo sistema operativo de red sobre uno ya existente, asegurando la integridad de la información y configurando las actualizaciones automáticas.

**Contenido**:

- Revisión de los procedimientos para realizar una instalación sobre un sistema operativo existente.
- Procedimientos para respaldar datos antes de la instalación.
- Herramientas para la verificación de compatibilidad del nuevo sistema operativo con el hardware y software actuales.
- Configuración de las actualizaciones automáticas de software y seguridad.

**Actividad**:  
Los estudiantes analizan un caso práctico donde deben instalar un nuevo sistema operativo de red sobre uno existente, asegurando que la información del servidor se mantenga intacta y configurando las actualizaciones automáticas.

**Evaluación**:  
Preguntas dirigidas sobre los procedimientos de respaldo y las opciones de configuración de actualizaciones automáticas.

**Materiales**:
Servidores con sistemas operativos de red existentes, manuales técnicos de sistemas operativos, herramientas de respaldo de datos.

### 🧰 Materiales necesarios

| Material | Cantidad | Uso en la clase |
|----------|----------|-----------------|
| Servidor con SO de red ya instalado | 1 por grupo | Analizar el caso práctico de instalación sobre sistema existente |
| Herramienta de verificación de compatibilidad del nuevo SO | 1 por equipo | Comprobar que el hardware y software actuales son compatibles con el nuevo SO |
| Software de respaldo del servidor | 1 por equipo | Generar un respaldo completo antes de proceder con la nueva instalación |
| Manuales técnicos de procedimientos de instalación paralela | 1 por grupo | Revisar métodos para proteger los datos durante la instalación |

:::note[🎟️ Ticket de salida]
1. ¿Qué riesgos específicos implica instalar un nuevo SO de red sobre uno ya existente, y cómo se mitiga cada uno?
2. ¿Por qué es necesario verificar la compatibilidad del nuevo SO con el hardware y software actuales antes de iniciar la instalación?
3. ¿Qué parte del análisis del caso práctico de hoy te resultó más compleja de planificar y por qué?
:::

---

### Clase 2: Instalación y Configuración del Nuevo Sistema Operativo de Red (6 horas adicionales)

**Objetivo**:  
Instalar un nuevo sistema operativo de red sobre uno ya existente, resguardando la integridad de la información y configurando las actualizaciones automáticas.

**Contenido**:

- Procedimientos para la instalación de un nuevo sistema operativo sobre un servidor con un sistema operativo existente.
- Métodos para asegurar la integridad de los datos durante la instalación (instalación paralela, particionado).
- Configuración de actualizaciones automáticas de software y seguridad.
- Solución de problemas comunes durante el proceso de instalación.

**Actividad**:  
Los estudiantes realizan la instalación de un nuevo sistema operativo de red sobre uno ya existente, configurando las actualizaciones automáticas y verificando la integridad de los datos.

**Evaluación**:  
Preinforme que documente el proceso de instalación, los problemas encontrados y las soluciones aplicadas para proteger la información y configurar las actualizaciones.

**Materiales**:
Servidores con sistemas operativos existentes, herramientas de instalación y configuración de sistemas operativos, manuales técnicos.

### 🧰 Materiales necesarios

| Material | Cantidad | Uso en la clase |
|----------|----------|-----------------|
| Servidor con SO existente y nuevo SO a instalar en paralelo | 1 por grupo | Realizar la instalación del nuevo SO preservando los datos del SO anterior |
| Medio de instalación del nuevo SO de red (USB/ISO) | 1 por grupo | Fuente de instalación del nuevo sistema operativo de red |
| Guía de configuración de actualizaciones automáticas del nuevo SO | 1 por grupo | Configurar actualizaciones de software y seguridad tras la instalación |
| Plantilla de preinforme de instalación sobre SO existente | 1 por estudiante | Registrar decisiones, problemas encontrados y soluciones aplicadas |

:::note[🎟️ Ticket de salida]
1. ¿Qué método de instalación (paralela o sobre la misma partición) elegirías para proteger los datos del servidor existente y por qué?
2. ¿Cómo verificarías que los datos del servidor anterior no fueron alterados ni perdidos tras instalar el nuevo SO de red?
3. ¿Qué decisión durante la instalación de hoy te generó más incertidumbre y por qué fue difícil tomarla?
:::

---

### Clase 3: Verificación y Documentación del Proceso de Instalación (12 horas)

**Objetivo**:  
Verificar que la instalación del nuevo sistema operativo de red sea exitosa, sin comprometer la información existente, y documentar todo el proceso.

**Contenido**:

- Verificación del funcionamiento del nuevo sistema operativo de red instalado.
- Pruebas de integridad de la información y compatibilidad con los sistemas existentes.
- Configuración final de las actualizaciones automáticas de software y seguridad.
- Elaboración de un informe técnico que documente el proceso de instalación y las pruebas realizadas.

**Actividad**:  
Los estudiantes realizan pruebas de verificación para asegurar la integridad del sistema operativo y la información después de la instalación, documentando todo el proceso en un informe técnico.

**Evaluación**:  
Informe técnico detallado que documente el proceso de instalación de un nuevo sistema operativo de red, la configuración de actualizaciones automáticas y las pruebas realizadas para asegurar la integridad de la información.

**Materiales**:
Servidores con sistemas operativos instalados, herramientas de diagnóstico, plantillas de informes técnicos.

### 🧰 Materiales necesarios

| Material | Cantidad | Uso en la clase |
|----------|----------|-----------------|
| Servidor con nuevo SO de red instalado y SO anterior preservado | 1 por grupo | Ejecutar pruebas de integridad de datos y funcionamiento del nuevo SO |
| Herramientas de verificación de integridad de datos (md5sum, rsync) | 1 por equipo | Comparar la integridad de los datos antes y después de la instalación |
| Checklist de configuración de actualizaciones automáticas | 1 por grupo | Verificar que las actualizaciones de software y seguridad están activas |
| Plantilla de informe técnico de instalación de SO sobre SO existente | 1 por estudiante | Documentar el proceso, pruebas de integridad y configuración final |

:::note[🎟️ Ticket de salida]
1. ¿Qué herramientas o comandos usarías para verificar la integridad de los archivos del servidor tras instalar el nuevo SO de red sobre el existente?
2. ¿Cómo confirmarías que las actualizaciones automáticas de seguridad del nuevo SO de red están correctamente configuradas y operativas?
3. ¿Qué aprendiste durante estas tres clases sobre la complejidad de instalar un SO sobre otro ya existente que no esperabas, y por qué te sorprendió?
:::

---

### Evaluaciones Finales para el Criterio 5.2

Al finalizar este criterio, los estudiantes deben entregar un informe técnico que documente el proceso de instalación de un nuevo sistema operativo de red sobre uno ya existente, asegurando la integridad de la información y configurando las actualizaciones automáticas. La evaluación se basará en la precisión del proceso y en la claridad de la documentación del mismo.
