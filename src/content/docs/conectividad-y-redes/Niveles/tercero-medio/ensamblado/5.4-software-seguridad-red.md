El estudiante debe aprender a instalar software de seguridad informática en redes de área local, configurando los parámetros de red del sistema operativo para garantizar la integridad de la información y aplicando reglas y métodos de respaldo de la información.



---

## Plan de Clases

### Clase 1: Introducción a la Seguridad Informática en Redes de Área Local (6 horas)

**Objetivo**:  
Entender los conceptos básicos de la seguridad informática en redes de área local, y cómo el software de seguridad contribuye a la protección de los datos.

**Contenido**:

- Principios de seguridad informática en redes LAN.
- Tipos de software de seguridad: antivirus, cortafuegos (firewalls), IDS/IPS (Sistemas de Detección y Prevención de Intrusiones).
- Configuración básica de los parámetros de red en sistemas operativos para asegurar la comunicación segura entre dispositivos.

**Actividad**:  
Los estudiantes investigan los diferentes tipos de software de seguridad informática y sus funcionalidades, y preparan un plan para la instalación de un software de seguridad en una red local.

**Evaluación**:  
Preguntas dirigidas sobre los conceptos clave de seguridad en redes locales y la importancia de configurar correctamente los parámetros de red.

**Materiales**:
Servidores o sistemas operativos de red, software de seguridad, manuales técnicos.

### 🧰 Materiales necesarios

| Material | Cantidad | Uso en la clase |
|----------|----------|-----------------|
| Servidor o equipo con SO de red para análisis | 1 por grupo | Investigar los tipos de software de seguridad aplicables a la red LAN |
| Software de seguridad de muestra (antivirus, firewall, IDS/IPS) | 1 por sala | Explorar las interfaces y funcionalidades de cada tipo de software |
| Manuales técnicos de seguridad en redes LAN | 1 por grupo | Revisar principios de seguridad y parámetros de red a configurar |
| Ficha de planificación de instalación de software de seguridad | 1 por grupo | Elaborar el plan de instalación del software para el caso práctico |

:::note[🎟️ Ticket de salida]
1. ¿Cuál es la diferencia entre un IDS y un IPS en el contexto de la seguridad de una red LAN, y cuándo conviene implementar cada uno?
2. ¿Por qué no es suficiente instalar solo un antivirus en los equipos de una red local para garantizar la seguridad de la información?
3. ¿Qué tipo de software de seguridad te resultó más desconocido hoy y por qué crees que es importante aprenderlo?
:::

---

### Clase 2: Instalación y Configuración de Software de Seguridad (6 horas adicionales)

**Objetivo**:  
Instalar software de seguridad informática en la red de área local, asegurando la protección de los datos y la correcta configuración de los parámetros de red.

**Contenido**:

- Instalación de software de seguridad (antivirus, cortafuegos, IDS/IPS) en servidores y dispositivos de la red.
- Configuración de reglas de seguridad en el sistema operativo, como el bloqueo de puertos o la configuración de permisos de acceso.
- Métodos de respaldo y recuperación de datos en redes locales.

**Actividad**:  
Los estudiantes instalan y configuran software de seguridad en un servidor de la red, aplicando reglas de seguridad y verificando la correcta protección de la información.

**Evaluación**:  
Preinforme que documente el proceso de instalación del software de seguridad y las configuraciones realizadas, incluyendo las reglas de protección implementadas.

**Materiales**:
Servidores con software de seguridad, herramientas de diagnóstico de red, manuales técnicos.

### 🧰 Materiales necesarios

| Material | Cantidad | Uso en la clase |
|----------|----------|-----------------|
| Servidor de red para instalar software de seguridad | 1 por grupo | Instalar y configurar antivirus, cortafuegos e IDS/IPS en la red LAN |
| Software de seguridad a instalar (antivirus, firewall, IDS/IPS) | 1 por equipo | Herramienta principal de la práctica de instalación y configuración |
| Guía de configuración de reglas del cortafuegos y permisos | 1 por grupo | Referencia para definir reglas de bloqueo de puertos y accesos |
| Plantilla de preinforme de instalación de software de seguridad | 1 por estudiante | Registrar configuraciones aplicadas y problemas encontrados |

:::note[🎟️ Ticket de salida]
1. ¿Qué reglas de cortafuegos configurarías en un servidor de red LAN escolar para protegerlo frente a accesos no autorizados desde internet?
2. ¿Cómo verificarías que las reglas de seguridad aplicadas en el cortafuegos no están bloqueando tráfico legítimo de los equipos de la red?
3. ¿Qué configuración de seguridad te resultó más compleja de implementar hoy y por qué fue difícil hacerlo correctamente?
:::

---

### Clase 3: Verificación y Documentación del Proceso de Instalación de Seguridad (12 horas)

**Objetivo**:  
Verificar que el software de seguridad instalado funcione correctamente en la red de área local y documentar todo el proceso de instalación y configuración.

**Contenido**:

- Pruebas funcionales para verificar la efectividad del software de seguridad en la protección de la red.
- Pruebas de respaldo y recuperación de datos para asegurar que el sistema sea capaz de mantener la integridad de la información.
- Elaboración de un informe técnico que documente el proceso de instalación del software de seguridad, las configuraciones aplicadas y los resultados de las pruebas de verificación.

**Actividad**:  
Los estudiantes realizan pruebas de verificación del software de seguridad en la red, asegurando que los dispositivos estén protegidos y que los datos sean respaldados correctamente. Posteriormente, documentan el proceso en un informe técnico.

**Evaluación**:  
Informe técnico detallado que documente el proceso de instalación, configuración y verificación del software de seguridad en la red de área local.

**Materiales**:
Servidores, sistemas operativos de red, herramientas de diagnóstico y recuperación de datos, plantillas de informes técnicos.

### 🧰 Materiales necesarios

| Material | Cantidad | Uso en la clase |
|----------|----------|-----------------|
| Servidor con software de seguridad instalado y configurado | 1 por grupo | Ejecutar pruebas de efectividad del antivirus, cortafuegos e IDS/IPS |
| Herramientas de diagnóstico de red y análisis de tráfico | 1 por equipo | Verificar que el software de seguridad detecta y bloquea amenazas correctamente |
| Escenario de prueba de respaldo y recuperación de datos | 1 por grupo | Confirmar que el sistema de respaldo protege la integridad de la información |
| Plantilla de informe técnico de seguridad en red LAN | 1 por estudiante | Documentar instalación, configuraciones aplicadas y resultados de las pruebas |

:::note[🎟️ Ticket de salida]
1. ¿Cómo probarías que el software antivirus instalado en el servidor detecta amenazas reales sin poner en riesgo la red durante la prueba?
2. ¿Qué información del proceso de instalación de seguridad debe quedar registrada en el informe técnico para que sea útil ante una auditoría de seguridad?
3. ¿Qué aprendiste sobre la seguridad informática en redes LAN que cambiará tu forma de proteger un sistema en el futuro, y por qué?
:::

---

### Evaluaciones Finales para el Criterio 5.4

Al finalizar este criterio, los estudiantes deben entregar un informe técnico que documente el proceso completo de instalación y configuración de software de seguridad en una red de área local, asegurando la integridad de la información y la correcta aplicación de reglas de seguridad. La evaluación se basará en la precisión de la instalación y en la claridad de la documentación del proceso.
