El estudiante debe aprender a verificar la compatibilidad del hardware del equipo con los requerimientos del sistema operativo a instalar, asegurando un buen funcionamiento.



---

## Plan de Clases

### Clase 1: Introducción a la Compatibilidad del Hardware con el Sistema Operativo (6 horas)

**Objetivo**:  
Entender los requisitos de compatibilidad entre el hardware y el sistema operativo, y aprender a identificar los componentes que necesitan verificación.

**Contenido**:

- Requisitos mínimos y recomendados del sistema operativo para el hardware (procesador, memoria, almacenamiento, etc.).
- Compatibilidad de controladores (drivers) con el sistema operativo.
- Herramientas y recursos en línea para verificar la compatibilidad del hardware (bases de datos de compatibilidad, sitios web de fabricantes, etc.).
- Problemas comunes de compatibilidad y cómo resolverlos.

**Actividad**:  
Los estudiantes investigan los requisitos de compatibilidad entre el hardware y el sistema operativo asignado, y crean una lista de los componentes que requieren verificación.

**Evaluación**:  
Preguntas dirigidas sobre los problemas de compatibilidad más comunes y las herramientas utilizadas para la verificación de hardware y drivers.

**Materiales**:
Computadoras con acceso a bases de datos de compatibilidad de hardware, fichas técnicas de sistemas operativos y componentes.

### 🧰 Materiales necesarios

| Material | Cantidad | Uso en la clase |
|----------|----------|-----------------|
| Computadores con acceso a bases de datos de compatibilidad | 1 por grupo | Investigación de requisitos de compatibilidad |
| Fichas técnicas de SO y componentes | 1 por grupo | Referencia para identificar incompatibilidades |
| Lista de componentes a verificar (plantilla) | 1 por estudiante | Actividad de identificación de requerimientos |
| Proyector | 1 | Presentación de herramientas de verificación en línea |

:::note[🎟️ Ticket de salida]

Antes de salir, cada estudiante responde individualmente (2–3 minutos):

1. ¿Por qué un **driver desactualizado** puede causar que un hardware funcione mal aunque sea compatible con el SO?
2. ¿Qué herramienta o sitio web usarías para verificar si una tarjeta de red es compatible con **Windows 11**?
3. ¿Qué componente del equipo asignado te genera más dudas sobre su compatibilidad y por qué?

:::

---

### Clase 2: Verificación de la Compatibilidad de Componentes de Hardware (6 horas adicionales)

**Objetivo**:  
Verificar la compatibilidad de los componentes de hardware con el sistema operativo previsto utilizando herramientas y recursos de diagnóstico.

**Contenido**:

- Verificación de la compatibilidad del procesador, memoria RAM, disco duro/SSD, tarjeta gráfica y otros periféricos con el sistema operativo.
- Uso de herramientas de verificación de controladores y soporte del sistema operativo.
- Resolución de problemas de compatibilidad detectados durante la verificación (actualización de BIOS, instalación de controladores alternativos, etc.).
- Configuración de las opciones necesarias para garantizar la compatibilidad, como cambios en el BIOS o actualizaciones de firmware.

**Actividad**:  
Los estudiantes seleccionan un conjunto de hardware y verifican su compatibilidad con el sistema operativo asignado utilizando herramientas de diagnóstico y recursos en línea.

**Evaluación**:  
Preinforme que documente el proceso de verificación de compatibilidad, detallando los componentes comprobados, las herramientas utilizadas y los resultados obtenidos.

**Materiales**:
Computadoras con acceso a herramientas de diagnóstico de hardware y bases de datos de compatibilidad de sistemas operativos, manuales técnicos.

### 🧰 Materiales necesarios

| Material | Cantidad | Uso en la clase |
|----------|----------|-----------------|
| Computadores con herramientas de diagnóstico y bases de compatibilidad | 1 por grupo | Verificación activa de compatibilidad de componentes |
| Conjunto de hardware seleccionado (de Clase 1) | 1 por grupo | Base para verificación de compatibilidad con el SO |
| Manuales técnicos de hardware y SO | 1 por grupo | Referencia para resolución de incompatibilidades |
| Plantilla de preinforme de verificación | 1 por estudiante | Registro de componentes verificados y resultados |

:::note[🎟️ Ticket de salida]

Antes de salir, cada estudiante responde individualmente (2–3 minutos):

1. Describe el proceso que seguiste para verificar que la **tarjeta gráfica** es compatible con el SO asignado.
2. ¿Qué solución aplicarías si el BIOS no reconoce un **SSD NVMe** en un equipo antiguo?
3. ¿Encontraste alguna incompatibilidad real hoy? ¿Cómo la resolviste o planeas resolverla?

:::

---

### Clase 3: Documentación del Proceso de Verificación y Resolución de Problemas (12 horas)

**Objetivo**:  
Documentar el proceso completo de verificación de compatibilidad de hardware con el sistema operativo, incluyendo las pruebas realizadas y las soluciones implementadas en caso de problemas.

**Contenido**:

- Elaboración de un informe técnico que documente el proceso de verificación de compatibilidad.
- Descripción de las herramientas utilizadas y los resultados de las pruebas de compatibilidad.
- Justificación de las decisiones tomadas para resolver problemas de compatibilidad encontrados durante la verificación.
- Verificación final del correcto funcionamiento del hardware y el sistema operativo tras resolver los problemas de compatibilidad.

**Actividad**:  
Los estudiantes elaboran un informe técnico que documente todo el proceso de verificación de compatibilidad de hardware con el sistema operativo, explicando los problemas encontrados y las soluciones aplicadas.

**Evaluación**:  
Informe técnico detallado que documente el proceso de verificación, las pruebas realizadas y los problemas resueltos, justificando las decisiones técnicas tomadas.

**Materiales**:
Computadoras con acceso a herramientas de verificación de compatibilidad, plantillas de informes técnicos, manuales de hardware y sistemas operativos.

### 🧰 Materiales necesarios

| Material | Cantidad | Uso en la clase |
|----------|----------|-----------------|
| Computadores con herramientas de verificación de compatibilidad | 1 por grupo | Pruebas finales y elaboración del informe |
| Notas y preinformes de Clases 1 y 2 | 1 por grupo | Base para el informe técnico final |
| Plantillas de informes técnicos | 1 por estudiante | Documentación del proceso completo de verificación |
| Proyector (opcional) | 1 | Presentación de resultados si se requiere |

:::note[🎟️ Ticket de salida]

Antes de salir, cada estudiante responde individualmente (2–3 minutos):

1. ¿Qué sección del informe técnico describe las **soluciones aplicadas** para resolver incompatibilidades?
2. ¿Por qué es importante documentar también los componentes que resultaron **compatibles**, no solo los problemáticos?
3. ¿Qué aprendiste sobre la compatibilidad de hardware que **cambiaría tu proceso** de selección en el futuro?

:::

---

### Evaluaciones Finales para el Criterio 2.3

Al finalizar este criterio, los estudiantes deben entregar un informe técnico que documente el proceso completo de verificación de compatibilidad de hardware con el sistema operativo, justificando las decisiones y describiendo los problemas resueltos. La evaluación se basará en la precisión del análisis de compatibilidad y la calidad de la documentación técnica.
