El estudiante debe aprender a actualizar un sistema operativo de red, de acuerdo con los requerimientos del usuario, utilizando los comandos y herramientas necesarias para verificar su correcto funcionamiento, garantizando la seguridad y la integridad del sistema.



---

## Plan de Clases

### Clase 1: Preparación para la Actualización del Sistema Operativo de Red (6 horas)

**Objetivo**:
Preparar el entorno para la actualización del sistema operativo de red, asegurando que todos los componentes estén listos y sean compatibles con la nueva versión del sistema operativo.

**Contenido**:

- Revisión de los requerimientos de hardware y software para la actualización del sistema operativo de red.
- Procedimientos de respaldo de datos y creación de puntos de restauración antes de la actualización.
- Herramientas y comandos para verificar la compatibilidad de los componentes y la versión del sistema operativo.

**Actividad**:
Los estudiantes verifican la compatibilidad del servidor y sus componentes con la nueva versión del sistema operativo de red, y preparan el entorno creando puntos de restauración y realizando respaldos de datos.

**Evaluación**:
Preguntas dirigidas sobre la importancia de los respaldos y la preparación previa para la actualización de un sistema operativo de red.

**Materiales**:
Servidores con sistemas operativos de red, herramientas de diagnóstico de hardware y manuales técnicos.

### 🧰 Materiales necesarios

| Material | Cantidad | Uso en la clase |
|----------|----------|-----------------|
| Servidor con SO de red instalado (Linux/Windows Server) | 1 por grupo | Verificar compatibilidad con la nueva versión y crear punto de restauración |
| Software de respaldo (backup) del servidor | 1 por equipo | Generar copia de seguridad completa antes de la actualización |
| Manuales técnicos del fabricante del SO de red | 1 por grupo | Revisar requisitos mínimos y procedimientos oficiales de actualización |
| USB o almacenamiento externo para respaldo | 1 por grupo | Almacenar la copia de seguridad generada antes de actualizar |

:::note[🎟️ Ticket de salida]
1. ¿Qué información del servidor debes respaldar obligatoriamente antes de actualizar el sistema operativo de red y por qué es crítico hacerlo?
2. ¿Cómo verificarías que el hardware del servidor es compatible con la nueva versión del SO de red antes de iniciar la actualización?
3. ¿Qué parte del proceso de preparación para la actualización del SO de red te generó más dudas hoy y por qué?
:::

### Clase 2: Actualización del Sistema Operativo de Red (6 horas adicionales)

**Objetivo**:
Realizar la actualización del sistema operativo de red utilizando comandos y herramientas especializadas, garantizando la correcta instalación y funcionalidad del sistema.

**Contenido**:

- Comandos y herramientas para la actualización del sistema operativo de red (Linux, Windows Server).
- Procedimientos de actualización manual y automática del sistema operativo de red.
- Solución de problemas comunes durante el proceso de actualización.

**Actividad**:
Los estudiantes actualizan el sistema operativo de red en un servidor asignado, utilizando comandos y herramientas para completar el proceso de manera eficiente.

**Ejemplos de comandos de actualización:**

**Ubuntu Server:**

```bash
# Actualizar lista de paquetes
sudo apt update

# Actualizar sistema completo
sudo apt upgrade -y

# Actualizar el kernel si es necesario
sudo apt dist-upgrade -y

# Verificar versión del sistema
lsb_release -a
```

**CentOS/RHEL:**

```bash
# Actualizar sistema
sudo yum update -y

# o en versiones más nuevas
sudo dnf update -y

# Verificar versión
cat /etc/redhat-release
```

**Evaluación**:
Preinforme que documente el proceso de actualización del sistema operativo, los problemas encontrados y las soluciones aplicadas.

**Materiales**:
Servidores de red, herramientas de actualización de sistemas operativos, manuales técnicos.

### 🧰 Materiales necesarios

| Material | Cantidad | Uso en la clase |
|----------|----------|-----------------|
| Servidor con SO de red desactualizado (Ubuntu Server o CentOS) | 1 por grupo | Ejecutar la actualización del sistema usando comandos apt/yum/dnf |
| Acceso a internet o repositorio local de paquetes | 1 por sala | Descargar paquetes de actualización del fabricante |
| Referencia de comandos de actualización del SO de red | 1 por estudiante | Guía rápida de comandos apt, yum y dnf para la actualización |
| Plantilla de preinforme de actualización del SO de red | 1 por estudiante | Registrar el proceso de actualización y los problemas encontrados |

:::note[🎟️ Ticket de salida]
1. ¿Cuál es la diferencia entre ejecutar `apt upgrade` y `apt dist-upgrade` en Ubuntu Server, y cuándo es necesario usar el segundo?
2. ¿Qué deberías verificar inmediatamente después de que el servidor reinicie al finalizar la actualización para confirmar que el proceso fue exitoso?
3. ¿Qué comando o paso de la actualización del SO de red te resultó menos intuitivo hoy y por qué crees que fue así?
:::

### Clase 3: Verificación y Documentación del Proceso de Actualización (12 horas)

**Objetivo**:
Verificar que la actualización del sistema operativo de red funcione correctamente y documentar todo el proceso, asegurando que los requerimientos del usuario se cumplan.

**Contenido**:

- Pruebas funcionales para verificar el correcto funcionamiento del sistema operativo actualizado.
- Pruebas de compatibilidad del software y hardware después de la actualización.
- Elaboración de un informe técnico que documente el proceso de actualización, los problemas resueltos y las pruebas de verificación.

**Actividad**:
Los estudiantes realizan pruebas de verificación del sistema operativo de red actualizado y documentan todo el proceso en un informe técnico.

**Evaluación**:
Informe técnico detallado que documente el proceso completo de actualización del sistema operativo de red, incluyendo las pruebas realizadas y las soluciones aplicadas.

**Materiales**:
Servidores con sistemas operativos de red, herramientas de diagnóstico, plantillas de informes técnicos.

### 🧰 Materiales necesarios

| Material | Cantidad | Uso en la clase |
|----------|----------|-----------------|
| Servidor con SO de red recién actualizado | 1 por grupo | Ejecutar pruebas funcionales y de compatibilidad post-actualización |
| Herramientas de diagnóstico del sistema (lsb_release, systemctl, journalctl) | 1 por equipo | Verificar la versión actualizada y el estado de los servicios del servidor |
| Checklist de pruebas post-actualización del SO de red | 1 por grupo | Guiar la verificación sistemática del sistema actualizado |
| Plantilla de informe técnico de actualización del SO de red | 1 por estudiante | Documentar el proceso completo, incidencias y resultados de las pruebas |

:::note[🎟️ Ticket de salida]
1. ¿Qué comandos usarías en Linux para confirmar la versión del SO de red instalada y verificar que todos los servicios críticos están activos tras la actualización?
2. ¿Cómo detectarías si una actualización del SO de red generó una regresión en algún servicio de red, como DNS o DHCP?
3. ¿Qué aprendiste sobre la importancia de documentar el proceso de actualización de un SO de red que cambió tu forma de ver este tipo de tareas, y por qué?
:::

---

### Evaluaciones Finales para el Criterio 5.1

Al finalizar este criterio, los estudiantes deben entregar un informe técnico que documente el proceso de actualización del sistema operativo de red, verificando la compatibilidad y estabilidad del sistema. La evaluación se basará en la precisión de la actualización y en la claridad en la documentación del proceso.
