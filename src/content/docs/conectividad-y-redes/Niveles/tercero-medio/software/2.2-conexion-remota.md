El estudiante debe aprender a establecer conectividad entre un equipo terminal y un servidor WEB y FTP, accediendo a los servicios de página WEB mediante HTTP/HTTPS y transferencia de archivos mediante FTP.



---

## Plan de Clases

### Clase 1: Conceptos Básicos de Conectividad entre Terminal y Servidor (6 horas)

**Objetivo**:  
Comprender los principios básicos de la conectividad entre un equipo terminal y un servidor WEB y FTP en una red LAN.

**Contenido**:

- Conceptos de cliente-servidor en una red: cómo funciona la conexión entre un terminal (cliente) y un servidor para acceder a servicios web y FTP.
- Protocolos de comunicación: HTTP, HTTPS y FTP, su funcionamiento y diferencias.
- Requisitos para establecer una conexión: dirección IP del servidor, puertos de red, permisos de acceso.

**Actividad**:  
Los estudiantes investigan cómo funciona la conectividad entre un equipo terminal y un servidor WEB y FTP, discutiendo los protocolos y configuraciones necesarias para acceder a los servicios en una red LAN.

**Evaluación**:  
Preguntas dirigidas sobre los conceptos de cliente-servidor, protocolos de comunicación (HTTP, HTTPS, FTP) y los requisitos para establecer conectividad.

**Materiales**:  
Computadoras con acceso a sistemas operativos de red, manuales de configuración de red, guías de protocolos de comunicación.

---

### Clase 2: Configuración de la Conectividad entre Terminal y Servidor (6 horas adicionales)

**Objetivo**:  
Configurar la conectividad entre un equipo terminal y un servidor WEB y FTP en un entorno LAN, asegurando que se puedan acceder a los servicios correctamente.

**Contenido**:

- Configuración de direcciones IP en el terminal y el servidor: asignación de IP estáticas o dinámicas, configuración de máscara de subred y puerta de enlace.
- Pruebas de conectividad a través de ping y herramientas de diagnóstico de red.
- Acceso a servicios web mediante HTTP/HTTPS: ingreso a la página web configurada en el servidor.
- Acceso y uso del servidor FTP: autenticación de usuarios, subida y descarga de archivos a través de un cliente FTP.

**Actividad**:  
Los estudiantes configuran un equipo terminal para conectarse a un servidor WEB y FTP, verificando que los servicios funcionen correctamente mediante la ejecución de pruebas de conexión y transferencia de archivos.

**Evaluación**:  
Preinforme donde los estudiantes describen el proceso de configuración del terminal y el servidor, así como los resultados obtenidos en las pruebas de acceso a los servicios WEB y FTP.

**Materiales**:  
Computadoras con sistemas operativos de red, software de servidor WEB y FTP, herramientas de diagnóstico de red (ping, traceroute), clientes FTP (FileZilla, WinSCP).

---

### Clase 3: Pruebas Avanzadas de Conectividad y Solución de Problemas (12 horas)

**Objetivo**:  
Realizar pruebas avanzadas de conectividad y solucionar problemas que afecten el acceso a los servicios WEB y FTP en un ambiente LAN.

**Contenido**:

- Pruebas avanzadas de conectividad: uso de herramientas de diagnóstico como traceroute, netstat y herramientas de monitoreo de tráfico de red.
- Resolución de problemas de conectividad: diagnóstico de problemas comunes como fallos en la resolución de DNS, problemas de firewall, configuración de puertos.
- Optimización de la conectividad: ajustar configuraciones para mejorar el rendimiento en la transferencia de archivos y el acceso a la web.

**Actividad**:  
Los estudiantes realizan pruebas avanzadas para verificar la conectividad entre el terminal y el servidor, identificando y resolviendo problemas de conexión. También optimizan las configuraciones de red para mejorar el rendimiento de los servicios WEB y FTP.

**Evaluación**:  
Informe técnico avanzado que documente las pruebas de conectividad realizadas, los problemas encontrados y las soluciones aplicadas, junto con las mejoras de rendimiento implementadas.

**Materiales**:
Computadoras con sistemas operativos de red, herramientas avanzadas de diagnóstico de red (Wireshark, traceroute, netstat), software de servidores WEB y FTP.

### 🧰 Materiales necesarios

| Material | Cantidad | Uso en la clase |
|---|---|---|
| Computadoras con sistema operativo de red (cliente y servidor) | 1 par por grupo | Establecer y verificar la conectividad entre terminal y servidor |
| Cliente FTP (FileZilla o WinSCP) | 1 por equipo | Realizar transferencias de archivos mediante FTP desde el terminal |
| Herramientas de diagnóstico de red (ping, traceroute, netstat) | 1 por equipo | Verificar la conectividad y diagnosticar problemas de conexión |
| Analizador de tráfico (Wireshark) | 1 por equipo | Observar el tráfico HTTP/HTTPS y FTP durante las pruebas de conectividad |

### Evaluaciones Finales para el Criterio 2.2

Al finalizar este criterio, los estudiantes deben entregar un informe técnico que documente el proceso de configuración del terminal y servidor, los resultados de las pruebas de conectividad entre los servicios WEB y FTP, y las soluciones implementadas para resolver cualquier problema. La evaluación se basará en la precisión de la configuración, la solución de problemas y la claridad en la documentación del informe.
