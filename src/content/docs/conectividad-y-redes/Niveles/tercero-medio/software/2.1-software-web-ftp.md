El estudiante debe aprender a instalar un software de aplicación para servicios WEB y FTP en un equipo computacional, verificando las configuraciones y capacidades del hardware para soportar la conectividad de usuarios en un ambiente de red LAN.



---

## ¿Por qué importa este criterio?

Instalar un servidor WEB o FTP no consiste solo en ejecutar un instalador. Si el hardware no soporta la carga, si los puertos no se configuran bien o si el servicio queda sin pruebas de acceso, la instalación puede parecer exitosa pero fallar en uso real.

> **Idea fuerza**: una instalación útil de servicios WEB y FTP combina tres cosas: software correcto, hardware suficiente y pruebas que demuestren que el servicio realmente funciona dentro de la LAN.

---

## 📚 Clase 1 — Qué requiere un servidor WEB o FTP para funcionar bien

**⏳ Duración:** 90 minutos

### 🧰 Materiales necesarios

| Material | Cantidad | Uso en la clase |
|----------|----------|-----------------|
| Computadoras con acceso a internet | 1 por estudiante | Investigar software servidor y revisar requisitos de hardware |
| Manuales de servidores WEB y FTP | 1 por estudiante | Identificar funciones, requerimientos y procedimientos de instalación |
| Guías de requisitos de hardware | 1 por grupo | Comparar capacidades del equipo con la carga esperada |
| Proyector o pantalla compartida | 1 por sala | Modelar la lectura técnica de requisitos y configuraciones |

### Distribución del tiempo

| Fase | Actividad | Tiempo |
|------|-----------|--------|
| 🟡 Inicio | Qué significa realmente “instalar un servicio” | 15 min |
| 🟠 Teoría | Función, requisitos y diferencias entre WEB y FTP | 20 min |
| 🔴 Modelado | Análisis guiado de software y hardware necesario | 15 min |
| 🟤 Práctica | Comparación técnica de opciones de servidor | 30 min |
| 🔵 Cierre | Síntesis de decisiones críticas | 10 min |

---

### 🟡 Fase 1 — Inicio: un servidor instalado no siempre es un servicio operativo (0–15 min)

**Propósito:** distinguir entre instalar un programa y dejar un servicio funcional para usuarios reales.

<details>
<summary>📋 Ver actividades de esta fase</summary>

1. Plantear un caso donde un servidor fue instalado, pero los usuarios no logran acceder bien al servicio.
2. Preguntar qué pudo haber fallado: software, puertos, permisos, recursos del equipo.
3. Listar en pizarra qué necesita realmente un servicio WEB o FTP para operar.
4. Presentar el objetivo del criterio: instalar, configurar y comprobar servicios WEB y FTP en un equipo LAN.

**Materiales:** proyector, pizarra y ejemplo de caso de instalación fallida.

</details>



Conviene pedir que el curso justifique por qué un servidor web y uno FTP no exigen exactamente lo mismo del equipo. Eso ayuda a conectar instalación de software con capacidad real del hardware y no tratar ambos servicios como si fueran equivalentes.



---

### 🟠 Fase 2 — Teoría: software, hardware y requisitos de servicio (15–35 min)

**Estrategia:** presentar los servicios WEB y FTP como soluciones técnicas con requerimientos concretos, no como aplicaciones genéricas.

<details>
<summary>🧠 Contenido o apoyo guiado</summary>

- Definición y función de servidores WEB (Apache, Nginx, IIS) y FTP (vsftpd, FileZilla Server).
- Requisitos mínimos de hardware y software para su implementación.
- Configuraciones básicas necesarias para que el servicio responda dentro de una red LAN.
- Diferencias entre exponer una página, transferir archivos y sostener múltiples usuarios.

</details>



No basta con que el software “abra”. Si el equipo no soporta la carga o el servicio no queda correctamente configurado, la instalación no cumple su objetivo técnico aunque no presente errores visibles al principio.



---

### 🔴 Fase 3 — Modelado: cómo decidir si el equipo y el software son adecuados (35–50 min)

**Propósito:** mostrar cómo se relacionan requerimientos del servicio con capacidades reales del hardware.

<details>
<summary>🎬 Secuencia de la demostración</summary>

1. Revisar especificaciones de un software servidor WEB o FTP.
2. Compararlas con RAM, CPU y almacenamiento de un equipo disponible.
3. Decidir si el equipo puede alojar el servicio de forma razonable.
4. Explicar qué ajustes mínimos deben verificarse antes de instalar.

</details>

---

### 🟤 Fase 4 — Práctica guiada: selección e instalación inicial del servicio (50–80 min)

**Organización:** grupos de 3 estudiantes.

<details>
<summary>📌 Caso de trabajo</summary>

Cada grupo recibe un equipo con una configuración determinada y debe decidir si es apto para instalar un servicio WEB o FTP en una red LAN. Luego debe justificar la elección del software y preparar la instalación inicial.

</details>

<details>
<summary>✍️ Pasos de la actividad práctica</summary>

| Paso | Actividad | Tiempo |
|------|-----------|--------|
| 1 | Revisar requisitos del servicio | 5 min |
| 2 | Verificar recursos del equipo | 10 min |
| 3 | Elegir software adecuado para el caso | 5 min |
| 4 | Preparar instalación y configuración inicial | 10 min |

</details>



Si un grupo se queda solo en comparar “software popular”, conviene pedirle que explique qué servicio necesita realmente el caso y qué recurso del equipo sería el cuello de botella más probable.



---

### ⚫ Fase 5 — Cierre y síntesis (80–90 min)

<details>
<summary>🎤 Actividades de cierre</summary>

1. Comparar dos decisiones de instalación y discutir cuál fue técnicamente más sólida.
2. Identificar un requisito de hardware que el curso subestimó al inicio.
3. Sintetizar qué hace que una instalación de servicio sea realmente viable.

</details>

:::note[🎟️ Ticket de salida]

1. ¿Qué requisitos mínimos de hardware debe cumplir un equipo para alojar un servidor WEB y FTP funcional en una red LAN? Justifica cada requisito.
2. ¿Qué diferencia importante existe entre instalar un servicio y verificar que realmente quedó operativo para los usuarios?
3. ¿Qué parte del análisis previo a la instalación te resultó más difícil y cómo la resolviste?

:::



Guardar la justificación técnica de hoy ayuda a que la instalación práctica no se convierta en una secuencia mecánica, sino en una decisión conectada con capacidad real del equipo y objetivo del servicio.

## 🛠️ Clase 2 — Instalación y configuración inicial de los servicios

**⏳ Duración:** 90 minutos  
**🎯 Objetivos:** instalar el servidor WEB o FTP, aplicar configuraciones básicas y verificar que el servicio quede operativo en la LAN.

<details>
<summary>📋 Secuencia de actividades</summary>

1. Ejecutar la instalación del software seleccionado.
2. Configurar usuarios, permisos, puertos y acceso básico.
3. Verificar el estado del servicio dentro de la red local.
4. Registrar configuraciones y resultados preliminares.

</details>

<details>
<summary>💻 Ejemplo o referencia de trabajo</summary>

Comparación entre una instalación que "termina sin errores" y una instalación realmente funcional, donde el servicio responde desde otro equipo de la LAN y sus permisos o carpetas están correctamente definidos.

</details>

**📂 Recursos necesarios:** sistemas operativos de red, software servidor, herramientas de verificación de hardware y conectividad.

---

## 🎤 Clase 3 — Verificación avanzada, optimización y cierre técnico

**⏳ Duración:** 90 minutos  
**🎯 Objetivos:** probar conectividad, ajustar seguridad o rendimiento y dejar evidencia técnica del servicio instalado.

<details>
<summary>📋 Secuencia de la clase</summary>

1. Ejecutar pruebas de acceso WEB o transferencia FTP.
2. Revisar rendimiento, seguridad básica y comportamiento con más de un usuario.
3. Aplicar ajustes necesarios al servicio o al equipo.
4. Documentar resultados y decisiones de optimización.

</details>

**📂 Recursos:** herramientas de monitoreo, pruebas de carga, certificados SSL/TLS y terminales cliente.

---

## 📊 Rúbrica de Evaluación

| Criterio | Excelente (5) | Bueno (4) | Aceptable (3) | Insuficiente (2) |
|---|---|---|---|---|
| Selección técnica | Elige software y equipo con justificación clara y adecuada | La decisión es correcta con ajustes menores | La justificación es parcial | La elección no responde bien al caso |
| Instalación y configuración | El servicio queda correctamente instalado y funcional | Funciona con pequeños ajustes pendientes | Funciona de forma parcial | El servicio no queda operativo |
| Verificación de conectividad | Prueba el servicio con criterio y registra resultados claros | Verifica la mayoría de los aspectos relevantes | La verificación es incompleta | No comprueba adecuadamente el servicio |
| Relación hardware-servicio | Explica claramente cómo el hardware impacta el servicio | La explicación es mayormente correcta | La relación se describe de forma superficial | No relaciona hardware y servicio con claridad |

---

### Evaluaciones Finales para el Criterio 2.1

Al finalizar este criterio, los estudiantes deben entregar un informe técnico que documente el proceso de instalación del servidor WEB y FTP, las configuraciones aplicadas, las pruebas de conectividad realizadas y la verificación de que el hardware es suficiente para sostener el servicio dentro de la red LAN. La evaluación se basará en la calidad de la instalación, la validez de las pruebas y la claridad de la justificación técnica.



Una instalación útil no es solo la que termina sin errores, sino la que deja un servicio realmente accesible, verificable y sostenido por un equipo capaz de responder al uso esperado en la red.
