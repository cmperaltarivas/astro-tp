El estudiante debe aprender a verificar la conectividad en una red de área local (LAN) y comprobar la correcta implementación del esquema de direccionamiento IP, utilizando herramientas de simulación y pruebas de conectividad.



---

## Plan de Clases

### Clase 1: Pruebas de Conectividad en Redes LAN (6 horas)

**Objetivo:**

Aprender a realizar pruebas de conectividad en una red LAN y a interpretar los resultados.

**Contenido:**

- Herramientas de diagnóstico de red: Ping, Traceroute, Netstat.
- Proceso para verificar la conectividad en una red LAN.
- Identificación de problemas de conectividad comunes (errores de configuración, problemas físicos, etc.).

**Actividad:**

Los estudiantes realizarán pruebas de conectividad en una red LAN simulada, utilizando herramientas de diagnóstico para identificar problemas.

**Evaluación:**

Revisión de las pruebas de conectividad realizadas, asegurando que los estudiantes comprendan cómo detectar y resolver problemas de conectividad.

**Materiales:**

- Computadoras
- Herramientas de simulación de redes
- Dispositivos de red

### Clase 2: Verificación del Esquema de Direccionamiento IP (6 horas adicionales)

**Objetivo:**

Comprobar que el esquema de direccionamiento IP implementado en la red LAN es funcional y eficiente.

**Contenido:**

- Verificación de la correcta asignación de direcciones IP en los dispositivos de la red.
- Comprobación de la comunicación entre los dispositivos de la red mediante pruebas de direccionamiento IP.
- Diagnóstico de problemas de configuración y conectividad relacionados con el esquema de direccionamiento.

**Actividad:**

Los estudiantes comprobarán el esquema de direccionamiento IP de una red LAN simulada, utilizando herramientas de simulación y pruebas de conectividad.

**Evaluación:**

Revisión de los resultados de las pruebas de direccionamiento, asegurando que las direcciones IP estén correctamente asignadas y que el esquema de direccionamiento sea eficiente.

**Materiales:**

- Computadoras
- Herramientas de simulación de redes
- Guías de direccionamiento IP

### Clase 3: Ajustes y Optimización del Esquema de Direccionamiento IP (12 horas)

**Objetivo:**

Optimizar el esquema de direccionamiento IP en una red LAN para mejorar su conectividad y rendimiento.

**Contenido:**

- Identificación de problemas en el esquema de direccionamiento IP mediante pruebas de conectividad.
- Ajustes y corrección de problemas de asignación de direcciones IP y subredes.
- Pruebas finales de conectividad para verificar los ajustes realizados.

**Actividad:**

Los estudiantes realizarán ajustes en el esquema de direccionamiento IP de una red LAN simulada, verificando las mejoras en la conectividad y el rendimiento de la red.

**Evaluación:**

Revisión de los ajustes realizados y las pruebas finales, asegurando que el esquema de direccionamiento IP esté optimizado y que los dispositivos de la red se comuniquen correctamente.

**Materiales:**

- Computadoras
- Software de simulación de redes
- Herramientas de diagnóstico de red

### Evaluaciones Finales para el Criterio 2.3

Al finalizar este criterio, los estudiantes deberán entregar un **informe técnico** que documente las pruebas de conectividad realizadas, el esquema de direccionamiento IP implementado y los ajustes efectuados. La evaluación se basará en la precisión del esquema de direccionamiento y la correcta verificación de la conectividad.

### 🧰 Materiales necesarios

| Material | Cantidad | Uso en la clase |
|----------|----------|-----------------|
| Computadoras con herramientas de diagnóstico (ping, traceroute, netstat) | 1 por estudiante | Ejecutar pruebas de conectividad y diagnosticar problemas |
| Software de simulación de redes (Packet Tracer/GNS3) | 1 por estudiante | Replicar la red LAN y verificar el esquema de direccionamiento |
| Guías de interpretación de resultados de pruebas de red | 1 por estudiante | Analizar salidas de ping y traceroute correctamente |
| Planilla de registro de pruebas de conectividad | 1 por estudiante | Documentar resultados, errores y correcciones realizadas |
